EOS Files¶

The framework contains functions to load thermal EOS from file. The type of EOS is determined transparently to the user. For this we defined a file format for thermal EOS, based on HDF5.

Loading EOS¶

Loading an EOS is simple, after including the header

#include "eos_thermal_file.h"

using namespace EOS_Toolkit;

one can obtain an EOS via

units u = units::geom_solar();
auto eos = load_eos_thermal("path/example.eos.h5", u);

The EOS files are based on SI units internally, therefore one has to specify the geometric unit system the returned EOS object should employ. In the example above, we use \(G=c=M_\odot=1\).

This version of the library comes with one example file EOS/HYB1.80_MS1_PP.eos.h5, which is a hybrid EOS based on the MS1 cold EOS.

Creating EOS Files¶

To create EOS files, the library provides a Python interface. The EOS file format should not matter to users unless they want to extend it with custom EOS types. The interface consists of a single-file module EOS/create/reprimand_eos_format.py. The following EOS types are available:

Hybrid EOS¶

To create a hybrid EOS with cold part given by a tabulated barotropic EOS, use the following. See Tabulated EOS for details on the cold EOS.

reprimand_eos_format.save_thermal_hybrid_table(path, name, cold_name, cold_comment, gamma_th, eps_max, poly_n, rho, eps, press, csnd, gm1, ye=None)¶

Create a file for a hybrid EOS based on a tabulated cold EOS

Parameters:

  • path – Path of the file to create. Ending should be .eos.h5

  • name – Name of the EOS

  • cold_name – Name of the underlying cold barotropic EOS

  • cold_comment – Description of underlying cold EOS

  • gamma_th – \(\Gamma_{th}\) defining the thermal part

  • eps_max – Maximum allowed specific internal energy \(\epsilon\)

  • poly_n – Adiabatic index used for extrapolating below lowest tabulated density

  • rho – Array with baryonic mass density \(\rho\, [\mathrm{kg}/\mathrm{m}^3]\) at the sample points

  • eps – Array with specific internal energy \(\epsilon\) [dimensionless] at sample points

  • press – Array with pressure \(P\,[\mathrm{Pa}]\) at sample points

  • csnd – Array with adiabatic sound speed \(c_s\,[\mathrm{m}/\mathrm{s}]\) at sample points

  • gm1 – Array with pseudo-enthalpy \(g-1\) [dimensionless] at sample points

  • ye – Optionally, array with electron fraction \(Y_e\) [dimensionless] at sample points.

To create a hybrid EOS with cold part given by a piecewise polytropic EOS, use the following. See Piecewise Polytropic EOS for details on the cold EOS.

reprimand_eos_format.save_thermal_hybrid_pwpoly(path, name, cold_name, gamma_th, eps_max, rho_poly, rho_max, rho_bnd, gammas, cold_comment=None)¶

Create a file for a hybrid EOS based on a piecewise polytropic cold EOS

Parameters:

  • path – Path of the file to create. Ending should be .eos.h5

  • name – Name of the EOS

  • cold_name – Name of the underlying cold barotropic EOS

  • gamma_th – \(\Gamma_{th}\) defining the thermal part

  • eps_max – Maximum allowed specific internal energy \(\epsilon\)

  • rho_poly – Polytropic density scale \(\rho_p\, [\mathrm{kg}/\mathrm{m}^3]\) of first segment

  • rho_max – Maximum allowed baryonic mass density \(\rho_\mathrm{max} \, [\mathrm{kg}/\mathrm{m}^3]\)

  • rho_bnd – Array with lower boundary \(\rho_{b,i}\, [\mathrm{kg}/\mathrm{m}^3]\) of baryonic mass densities for each segment. Must start at zero.

  • gammas – Array with adiabatic exponents \(\Gamma_i\) for each segment

  • cold_comment – Optionally, additional description of the cold EOS.

To create a hybrid EOS with cold part given by a polytropic EOS, use the following. See Polytropic EOS for details on the cold EOS.

  • n_adiab – Adiabatic index \(n\)
